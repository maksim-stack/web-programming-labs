# 📘 Лабораторна робота №10
## Тема: Аутентифікація у React-застосунку (JWT)
### 🎯 Мета роботи

Реалізувати frontend-частину застосунку з аутентифікацією на основі JWT-токена.
Забезпечити реєстрацію, логін, збереження токена, отримання даних користувача та захищені маршрути.
---
### 🛠️ Використані технології
- React (Vite)
- React Router DOM
- Axios
- JWT (JSON Web Token)
- LocalStorage
- Context API
- Node.js backend (NestJS з ЛР9)

### 🔐 Основна логіка
1. Реєстрація
- POST `/auth/register`
- створює нового користувача
- після успіху → перенаправлення на `/login`

---

2. Логін
- POST `/auth/login`
- сервер повертає JWT токен
- токен зберігається у localStorage
- викликається `/auth/me` для отримання користувача

---

3. Отримання користувача
- GET `/auth/me`
- виконується з заголовком:
- Authorization: Bearer <token>

---

4. Захищені маршрути
- `/profile` доступний лише авторизованим користувачам
- якщо користувач не авторизований → редирект на `/login`

---

5. Глобальний стан (AuthContext)

**Містить:**

- `user` — поточний користувач
- `loading` — стан завантаження
- `login()` — авторизація
- `register()` — реєстрація
- `logout()` — вихід із системи

---

6. Axios інтерсептор

-автоматично додає JWT у кожен запит
-обробляє 401 помилки

### 🔄 Сценарії роботи
- Реєстрація
1) Користувач вводить `email/password`
2) Дані відправляються на сервер
3) Успіх → редирект на `/login`
- Логін
1) Ввід `email/password`
2) Отримання JWT
3) Збереження токена
4) Запит `/auth/me`
5) Перехід на `/profile`

- Захищена сторінка
1) перевіряється наявність user
2) якщо немає → редирект на `/login`

- Логаут
1) очищення localStorage
2) скидання стану користувача
3) редирект на `/login`

---

### 🧠 Особливості реалізації
- Використано Context API для глобального стану
- JWT зберігається у localStorage
- Використано Axios для HTTP-запитів
- Реалізовані protected routes
- Стан аутентифікації зберігається після перезавантаження сторінки
