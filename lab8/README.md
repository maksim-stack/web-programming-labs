# Лабораторна робота №8  
## Інтеграція бази даних у Nest.js застосунок

---

## 📌 Опис проєкту.

REST API застосунок на NestJS з інтеграцією PostgreSQL через TypeORM.  
Реалізовано CRUD для задач та тегів, а також зв’язок ManyToMany.

---

## ⚙️ Технології

- NestJS
- TypeORM
- PostgreSQL
- Docker
- Node.js 18+
- class-validator

---

## 🐳 Запуск PostgreSQL (Docker)

```bash
docker run --name postgres-lab8 \
-e POSTGRES_USER=nestuser \
-e POSTGRES_PASSWORD=123456 \
-e POSTGRES_DB=tasks_db \
-p 5432:5432 \
-d postgres:16
```

## ⚙️ Налаштування .env
```.env
DB_HOST=localhost
DB_PORT=5432
DB_USERNAME=nestuser
DB_PASSWORD=123456
DB_NAME=tasks_db
```

## 🚀 Запуск проєкту
```bash
npm install
npm run migration:run
npm run start:dev
```

## 🧱 Міграції
```bash
npm run migration:generate
npm run migration:run
npm run migration:revert
```

## Контрольні запитання
### 1. Що таке ORM і яку проблему вона вирішує?

ORM (Object-Relational Mapping) — це технологія, яка дозволяє працювати з базою даних через об’єкти мови програмування замість SQL-запитів.

👉 Вирішує проблему:

- складності написання SQL
- різниці між об’єктною моделлю (код) і реляційною (таблиці)
- ручного мапінгу даних

---
### 2. Що таке Entity у TypeORM і як вона пов’язана з таблицею?

Entity — це клас у TypeScript, який описує структуру таблиці в базі даних.

👉 Кожен об’єкт цього класу = один рядок таблиці.

👉 Кожне поле класу = колонка таблиці.

---
### 3. Що таке Repository і для чого він використовується?

Repository — це спеціальний об’єкт TypeORM для роботи з конкретною Entity.

Він використовується для:

- створення записів (create/save)
- читання (find/findOne)
- оновлення (update)
- видалення (delete)

👉 Це головний інструмент доступу до БД у сервісах.

---
### 4. Що таке міграція і чим вона відрізняється від synchronize: true?

Міграція — це версіонований файл змін структури бази даних (створення/зміна таблиць).

Відмінність:
- migrations → контрольовані зміни, можна відкотити (safe)
- synchronize: true → автоматично змінює БД при старті (небезпечний, може видалити дані)

👉 У продакшені використовують тільки міграції.

---
### 5. Що таке junction-таблиця і коли вона створюється?

Junction table (проміжна таблиця) — це таблиця, яка створюється для реалізації зв’язку ManyToMany.

👉 Створюється автоматично ORM, коли:

- одна Task має багато Tag
- один Tag має багато Task

Вона містить:

- id задачі
- id тегу

👉 Використовується для зберігання зв’язків між сутностями.
